# O que é o HTML?

A linguagem HTML é simplesmente uma linguagem de Marcação de Hipertexto. Você irá utilizá-la para estruturar todo o conteúdo de sua página em qualquer software desenvolvido. Ela está exclusivamente no Front-end (basicamente tudo aquilo que o usuário consegue visualizar, ouvir etc.). Na história do HTML, houve diversas versões e atualmente estamos utilizando o HTML "SEMÂNTICO". Utilizamos "tags semânticas" para criar códigos em HTML para estruturar o **SEO** de cada sistema da melhor forma possível.

# O que é o CSS?

A linguagem CSS significa Folhas de Estilo em Cascata. Você irá utilizá-la apenas para poder estilizar páginas de qualquer site, organizando e estilizando todo o seu conteúdo HTML. O CSS é focado na interface de softwares; diferente de outras linguagens, de programação ou não, ela não foi marcada pelas suas versões, porém ao longo do tempo foram adicionadas a ela funcionalidades específicas, onde saímos de apenas ---> Estilizações básicas de documentos <--- para ---> Recursos avançados e automação <---. Hoje em dia ainda é muito utilizada em sites e sistemas, porém hoje já existem frameworks que ajudam a agilizar o desenvolvimento, fazendo com que o trabalho seja muito mais rápido do que usar apenas a linguagem pura. De qualquer forma, ela e também o HTML são linguagens básicas e supernecessárias para o aprendizado inicial do Front-end.

# O que é o JavaScript (JS)?

Diferente das duas linguagens anteriores (HTML e CSS), o JavaScript é uma linguagem de programação e por isso não se limita apenas a ser uma linguagem específica do Front-end, mas também é altamente recomendada no Back-end. Em algumas ocasiões, como em sistemas de tempo real, ela se mostra tão poderosa quanto diversas linguagens de programação específicas para Back-end na resolução de problemas. No Front-end, a linguagem JS é responsável principalmente por dar vida, interatividade e dinamismo às páginas web, mas a tecnologia JS não se limita apenas a softwares web.

# Quando, por que e como utilizar essas linguagens?

## HTML e CSS

Ambas as linguagens devem ser utilizadas no Front, pois são elas que cuidam de tudo o que o usuário interage visualmente. Enfim, você irá utilizá-las para criar estruturas de páginas com diversos tipos de conteúdos e, além disso, irá organizá-los e estilizar todo esse conteúdo.

## JAVASCRIPT

A linguagem JS, por mais que não seja exclusiva do Front, é extremamente utilizada e importante nele; com isso, ela é responsável por dar vida, interatividade e dinamismo às páginas, manipulando o HTML e também o CSS através de seus scripts, que podem ser incorporados dentro dos arquivos com extensão ".html" de forma interna ou de forma externa, que seria em arquivos com a extensão ".js".

# Introdução ao desenvolvimento de softwares (SEMÂNTICA E Sintaxe)

## HTML --- Tags semânticas

### SEMÂNTICA

Antes de iniciarmos na definição do que são tags semânticas, primeiro precisamos saber o que é exatamente a palavra "semântica" e o que de fato ela significa.

Enfim, sem enrolações, o significado da palavra "semântica" nada mais é do que dar Significado a algo. Sim, o significado da palavra "SEMÂNTICA" é literalmente dar significado a algo. Ou seja, quando adiciono a tag HEADER nos meus códigos, o que estou dizendo é que eu estou criando um Cabeçalho no meu HTML. O HTML5 trouxe tags de estrutura como header, main e footer que já contêm significado próprio. A principal mudança dessa versão foi literalmente a incrementação da semântica na maioria de suas tags para ajudar o Google e os navegadores a entenderem o site.

### SINTAXE 

Bom, mesmo que semântica e sintaxe **pareçam** semelhantes, a sintaxe tem **algo** bem diferente. E antes de saber como, onde e quando devemos aplicar a sintaxe corretamente, vamos primeiro, obviamente, saber o significado (a semântica) dessa palavra e depois começar a **aplicá-la** em nossos pequenos sistemas.

Sintaxe na programação é um conjunto de regras gramaticais que define a estrutura, pontuação e ordem correta dos símbolos e palavras das linguagens de programação de modo geral. 
Para aprender a sintaxe de uma linguagem você deve focar na aprendizagem através da escrita de códigos da linguagem escolhida, mas sempre evitando a memorização mecânica como forma de aprendizado verdadeiro sobre a  sintaxe. 

Você precisa estudar os seguintes assuntos que contém conexão direta com a sintaxe da linguagem do momento:

- **Escreva Código, Não Apenas Leia:** A memorização não funciona para a maioria; você deve digitar o código para entender como ele funciona.

- **Aprenda Estruturas Fundamentais:** Foque em entender como estruturas de controle (`if-else`), laços de repetição (`while`, `for`) e tipos de dados são implementados, como sugerido por recursos como o [CodeGym](https://codegym.cc/pt/groups/posts/pt.484.sintaxe-java-uma-breve-introducao-a-linguagem-de-programacao).

- **Prática Consciente (Projetos):** Tente construir pequenos projetos, dividindo o aprendizado em tópicos.

- **Use Plataformas de Desafios:** Utilize plataformas como o HackerRank para praticar a sintaxe de linguagens populares como Python, Java e C++.

- **Consulte a Documentação e Tutoriais:** Utilize bons tutoriais como o W3Schools para aprender a sintaxe básica, complementando com artigos e fóruns.

- **Repetição e Leitura Técnica:** Aprender sintaxe é como aprender uma língua estrangeira, exigindo leitura técnica constante, diz o [Medium](https://medium.com/codesigners-cc/designer-entenda-o-que-%C3%A9-sintaxe-e-fun%C3%A7%C3%A3o-no-mundo-da-programa%C3%A7%C3%A3o-ef4a35bc5433).

- **Roteiro de Estudos:** Siga um roteiro de estudos, como o indicado pela [DevMedia](https://www.devmedia.com.br/4-passos-para-aprender-a-programar-em-qualquer-linguagem/43126), que inclui aprender a lógica, paradigmas, boas práticas e frameworks. 

**Dica chave:** Não tente memorizar toda a sintaxe de uma vez; aprenda ciclicamente e aumente a complexidade gradualmente.

# Continuando em códigos HTML

Agora vamos pôr a mão na massa! Mas antes, peço para que vejam o código inicial básico de qualquer página HTML e a explicação de qual a responsabilidade de cada trecho. Isso é o básico para vocês compreenderem o que vou explicar adiante.

Começando a falar sobre tags, focaremos naquelas que eu denomino como as **principais tags semânticas**. Elas são: `header`, `main` e `footer`.

## Explicação das Tags

### 1. Tag `<header>` (Cabeçalho)

A tag **header** tem como objetivo dizer que aquele conteúdo faz parte do cabeçalho da página. Dentro dessa tag, podemos adicionar alguns tipos de conteúdos específicos, como:

- Logotipo e Nome do site/empresa;
    
- Menu de navegação principal;
    
- Links funcionais e Formulários de pesquisa;
    
- Chamadas para Ação (CTAs);
    
- Títulos e subtítulos;
    
- Ícones de redes sociais e Seletores de idioma.
    

### 2. Tag `<main>` (Conteúdo Principal)

Já a tag **main**, ela tem o objetivo de dizer que tudo o que está ali dentro é, de fato, o **conteúdo principal** do site. É o "coração" da página. Dentro dela, podemos colocar:

- **Conteúdo Central:** O texto, imagens e vídeos que explicam o tópico principal do documento.
    
- **Tags de Estrutura:** É comum organizar o conteúdo interno com tags como `<section>`, `<article>`, `<details>` e `<summary>`.
    
- **Elementos Textuais:** Tags de títulos (`<h1>` a `<h6>`), parágrafos (`<p>`), listas e links.
    
- **Mídia e Formulários:** Imagens, vídeos, áudios e formulários interativos que pertençam àquela página específica.
    

### 3. Tag `<footer>` (Rodapé)

Por último, iremos falar sobre a tag **footer**, que significa "Rodapé". Ela é usada para o final de uma seção ou da página inteira. Geralmente, ela contém metadados e informações auxiliares, como:

- **Informações de Copyright:** Avisos de direitos autorais e o ano de criação (ex: © 2026).
    
- **Dados de Contato:** Endereços, e-mails ou números de telefone (geralmente dentro da tag `<address>`).
    
- **Links Secundários:** "Termos de Uso", "Política de Privacidade", mapa do site ou redes sociais.
    
- **Navegação Auxiliar:** Menus simplificados que ajudam o usuário no fim da página.
    
- **Créditos:** Informações sobre o autor do conteúdo ou desenvolvedores do site.

### PRINCIPAIS TAGS HTML...

1. **`<article>` (O Artigo)** Essa é a tag principal de um blog. Ela deve envolver cada postagem. **Semântica:** Indica que o conteúdo dentro dela é independente. Se você "recortar" o artigo e colar em outro site, ele ainda faz sentido sozinho. **Uso:** Use para o corpo do post na página de leitura e também para os pequenos resumos na página inicial.
    
2. **`<section>` (Seção)** Serve para agrupar conteúdos que têm o mesmo tema dentro de uma página ou de um artigo. **Semântica:** Indica uma "divisão temática". **Exemplo:** Dentro de um post, você pode ter uma `<section>` para "Introdução", outra para "Desenvolvimento" e outra para "Conclusão".
    
3. **`<aside>` (Barra Lateral)** É utilizada para conteúdos que são relacionados ao tema principal, mas que não fazem parte dele diretamente. **Semântica:** Significa "à parte" ou "ao lado". **Uso em Blogs:** Ideal para barras laterais com links de "Posts mais lidos", "Categorias", "Sobre o Autor" ou anúncios.
    
4. **`<nav>` (Navegação)** Deve envolver os conjuntos de links de navegação. **Semântica:** Avisa ao navegador: "Estes são os caminhos para andar pelo site". **Uso:** Dentro do header para o menu principal ou no footer para links de redes sociais.
    
5. **`<time>` (Data e Hora)** Essencial para blogs, pois ajuda o Google a saber quando o post foi publicado. **Semântica:** Representa uma data específica no calendário ou um horário. **Exemplo:** `<time datetime="2026-04-13">13 de Abril</time>`

### EXPLICAÇÃO DAS TAGS BÁSICAS DE UMA PÁGINA HTML

```
<html lang="pt-br">
<!-- A tag raiz que envolve todo o conteúdo do site. 'lang' define o idioma como português -->

<head>
    <!-- Área de metadados: informações que não aparecem na página, mas são importantes -->
    
    <meta charset="UTF-8">
    <!-- Define a codificação de caracteres como UTF-8, essencial para exibir acentos corretamente -->
    
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <!-- Faz o site se ajustar automaticamente ao tamanho da tela (responsivo) -->
    
    <title>Título da Minha Página</title>
    <!-- Define o nome da página que aparece na aba do navegador -->
</head>

<body>
    <!-- Área de conteúdo visível: Tudo o que o usuário vê na tela -->

    <h1>Este é o meu título principal</h1>
    <!-- Título nível 1: Usado para o título principal da página -->

    <p>Este é um parágrafo básico de texto.</p>
    <!-- Tag para criar um parágrafo -->

    <a href="https://google.com">Este é um link</a>
    <!-- Tag âncora: cria um link para outro site -->

    <!-- Você pode colocar comentários também no meio do código -->
    
</body>
<!-- Fim da tag body -->

</html>
```
